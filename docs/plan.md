# stillne — 企画書

インテリア・雑貨ECショップ「stillne」のプロジェクト企画書。
Medusa v2 + Next.js（App Router）で構築するミニマルなオンラインストア。

---

## 1. プロジェクト概要

### 1.1 コンセプト

**プロジェクト名:** stillne（シャドウアール）
**由来:** shadow（影・陰影）+ r（realm / refined）。光と影のコントラストが生む洗練された空間を表現する。

**コンセプト:** 「静かに佇む、美しい日用品」
日常に溶け込みながらも、手に取るたびに小さな満足感を与えるインテリア・雑貨を届けるオンラインストア。過剰な装飾を排し、素材と形状の美しさで語るプロダクトをキュレーションする。

**提供する価値:**
- 厳選されたミニマルデザインのインテリア・雑貨
- 商品の質感や空間での佇まいが伝わるビジュアル体験
- ストレスのない、直感的な購買フロー

### 1.4 デモデータ方針

本プロジェクトは開発・デモ段階のため、すべてのコンテンツはプレースホルダーで構成する。
本番運用時にコンテンツを差し替えやすい設計とすること。

#### 画像
- すべての商品画像・バナー画像にプレースホルダーを使用する
- プレースホルダーは「ライトグレー背景（#F3F4F6）+ 中央にアイコン + サイズ表記」で統一する
- 実装: 専用の `PlaceholderImage` コンポーネントを用意し、`width`, `height`, `label` を受け取る
- 商品サムネイル: 480×480px、商品詳細ギャラリー: 800×800px、ヒーローバナー: 1920×600px

```
┌─────────────────────┐
│                     │
│     📷 アイコン      │
│   480 × 480        │
│   商品画像          │
│                     │
└─────────────────────┘
```

- Medusa側の商品登録時も、プレースホルダー画像ファイル（SVGまたはPNG）をアップロードする
- 将来的に実画像に差し替える際は、Medusa Admin UIから画像を更新するだけで反映される

#### テキスト
- 商品名: カテゴリに基づいたダミー名（例: 「セラミックフラワーベース A-001」）
- 商品説明: Lorem ipsumではなく、雰囲気の伝わる日本語ダミーテキストを使用する
- 価格: カテゴリごとに現実的な価格帯で設定する（¥1,500〜¥30,000）
- ダミーテキスト例: 「シンプルなフォルムが空間に溶け込む、日常使いのためのプロダクト。素材の質感を活かした仕上げが特徴です。」

#### シードデータ
- 開発用に20〜30点のダミー商品をシードスクリプトで投入する
- カテゴリごとに5〜8点ずつ、バリアント（カラー/サイズ）付きで登録する
- シードスクリプト: `scripts/seed.ts` に配置し、`npm run seed` で実行可能にする

### 1.2 ターゲットユーザー

**プライマリユーザー:**
- 25〜40代、都市部在住
- インテリアや暮らしの道具に関心が高い
- 「量より質」の価値観を持ち、長く使えるものを選びたい
- InstagramやPinterestでインテリア情報を収集している

**セカンダリユーザー:**
- ギフトを探している人（引越し祝い、誕生日等）
- インテリアショップの実店舗に足を運ぶ時間がない忙しい層

**ユーザーストーリー:**

1. 「リビングに置く花瓶を探している。シンプルで、どんなテーブルにも合うものが欲しい。カテゴリから"フラワーベース"を選んで、素材や色味を確認してから購入したい。」

2. 「友人の新居祝いに雑貨を贈りたい。予算は5,000〜10,000円。トップページのおすすめや特集から良さそうなものを見つけて、そのままギフトとして購入したい。」

3. 「以前購入したキャンドルホルダーが気に入ったので、同じブランドの他の商品も見たい。マイページの注文履歴から、そのブランドの商品一覧に飛びたい。」

### 1.3 ビジネスモデル

**収益モデル:** 商品販売（セレクトショップ型）
**想定価格帯:** ¥1,500〜¥30,000（中心価格帯: ¥3,000〜¥10,000）
**差別化ポイント:**
- 世界観を重視したビジュアルとUI/UX
- 厳選されたキュレーション（多品種少量ではなく、少品種を丁寧に紹介）
- 商品の「空間での見え方」を伝える写真・コンテンツ

**購買フロー（メインフロー）:**
```
トップページ → 商品一覧 → 商品詳細 → [カートに追加]
                                          ↓
                                      カートページ → チェックアウト → 注文完了
```

**アカウントフロー:**
```
ログイン / 新規登録 → マイページ → 注文履歴 → 注文詳細
                       ↓
                    プロフィール編集
```

```
任意のページ → [ヘッダーのカートアイコン] → カートドロワー → カートページ
```

## 3. 機能要件

### 3.1 フロントエンド（ストアフロント）

#### トップページ
| 項目 | 内容 |
|---|---|
| 概要 | ブランドの世界観を伝えるランディングページ |
| 優先度 | P0（必須） |
| Medusa API | `GET /store/products`（新着）, `GET /store/collections`（特集） |

**機能詳細:**
- ヒーローバナー: フルワイドのビジュアル + キャッチコピー + CTAボタン
- 新着商品セクション: 最新の商品を4〜8件表示（ProductCard）
- 特集セクション: コレクション単位でキュレーションした商品群
- カテゴリ導線: 主要カテゴリへの視覚的なリンク

#### 商品一覧
| 項目 | 内容 |
|---|---|
| 概要 | 商品をグリッド表示し、フィルタ・ソートで絞り込み |
| 優先度 | P0（必須） |
| Medusa API | `GET /store/products`, `GET /store/product-categories` |

**機能詳細:**
- グリッド表示（モバイル: 2列、デスクトップ: 3〜4列）
- カテゴリフィルター（サイドバー or ドロップダウン）
- ソート（新着順 / 価格昇順 / 価格降順）
- ページネーション（無限スクロール or ページ番号）
- 商品カードにはサムネイル、商品名、価格を表示

#### 商品詳細
| 項目 | 内容 |
|---|---|
| 概要 | 商品の詳細情報と購入アクション |
| 優先度 | P0（必須） |
| Medusa API | `GET /store/products/:id` |

**機能詳細:**
- 商品画像ギャラリー（メイン画像 + サムネイル切り替え）
- 商品名、価格、説明文
- バリアント選択（カラー / サイズ等のオプション）
- 在庫状態の表示（在庫あり / 残りわずか / 在庫切れ）
- 「カートに追加」ボタン
- 関連商品セクション（同カテゴリの商品）

#### カート
| 項目 | 内容 |
|---|---|
| 概要 | カート内容の確認・編集 |
| 優先度 | P0（必須） |
| Medusa API | `GET /store/carts/:id`, `POST/DELETE /store/carts/:id/line-items` |

**機能詳細:**
- カートドロワー: ヘッダーのアイコンクリックで表示するスライドパネル
- カートページ: 詳細な一覧表示
- 数量変更（+/- ボタン）
- アイテム削除
- 小計・送料・合計金額の表示
- 「チェックアウトへ進む」ボタン

#### チェックアウト
| 項目 | 内容 |
|---|---|
| 概要 | 配送先・決済情報の入力と注文確定 |
| 優先度 | P0（必須） |
| Medusa API | Cart API + Payment API + `POST /store/carts/:id/complete` |

**機能詳細:**
- ステップ形式: 配送先情報 → 配送方法 → 決済 → 確認
- 配送先住所フォーム（日本の住所形式対応）
- 配送方法選択（Medusa Fulfillment Module）
- Stripe Elementsによるカード決済フォーム（テストモード専用）
- 注文内容の最終確認画面
- 注文完了画面（注文番号、確認メール案内、**デモ注文の旨を明示**）

#### ユーザー認証・マイページ
| 項目 | 内容 |
|---|---|
| 概要 | 会員登録・ログイン・アカウント管理 |
| 優先度 | P1（重要） |
| Medusa API | Auth API + `GET /store/customers/me` |

**機能詳細:**
- メールアドレス + パスワードによる登録・ログイン（Medusa標準認証）
- プロフィール編集（名前、メールアドレス）
- 配送先住所の管理（追加・編集・削除）
- 注文履歴一覧と各注文の詳細表示
- ログアウト

#### 検索・フィルター
| 項目 | 内容 |
|---|---|
| 概要 | 商品検索とカテゴリフィルタリング |
| 優先度 | P1（重要） |
| Medusa API | `GET /store/products?q=`, `GET /store/product-categories` |

**機能詳細:**
- ヘッダーの検索バー（キーワード検索）
- 検索結果ページ
- カテゴリによるフィルター
- URLクエリパラメータによる状態管理（共有・ブックマーク可能）

### 3.2 管理画面（Medusa Admin UI + カスタムウィジェット）

#### 標準機能（Medusa Admin UI提供）

以下の機能はMedusa Admin UIが標準で提供するため、カスタム開発は不要。

| 機能 | 概要 |
|---|---|
| 商品管理 | 商品の登録・編集・削除、バリアント管理、画像アップロード、カテゴリ・コレクション管理 |
| 注文管理 | 注文一覧、注文詳細、出荷処理、返品・交換処理 |
| 顧客管理 | 顧客一覧、顧客詳細、顧客グループ |
| 在庫管理 | 在庫レベルの確認と更新、在庫拠点管理 |
| 割引・プロモーション | クーポンコード作成、プロモーションルール設定 |
| リージョン・配送 | リージョン設定、通貨設定、配送オプション管理 |
| 設定 | ストア設定、ユーザー管理、API Key管理 |

#### カスタムウィジェット（4機能）

Medusa Admin UIの標準的な見た目・ナビゲーションを維持したまま、ウィジェット（カード型パネル）として追加する。

##### ① 売上・注文ダッシュボード
| 項目 | 内容 |
|---|---|
| 配置場所 | Admin UIトップページ（Overview）にウィジェットとして追加 |
| 優先度 | P2（あると良い） |
| 使用API | `GET /admin/orders`（集計用） |

**表示内容:**
- 売上サマリーカード（今日 / 今週 / 今月の売上金額）
- 注文数の推移グラフ（折れ線チャート、直近30日間）
- 注文ステータス内訳（円グラフ: 新規 / 処理中 / 出荷済み / 完了 / キャンセル）
- 直近の注文5件のクイックリスト

**実装方針:**
- グラフライブラリ: Recharts（軽量・React対応）
- データはAdmin APIから取得し、フロント側で集計する
- デモ段階ではシードデータに基づくダミー集計を表示

##### ② 在庫アラート
| 項目 | 内容 |
|---|---|
| 配置場所 | Admin UIトップページ + 商品一覧ページにウィジェットとして追加 |
| 優先度 | P2（あると良い） |
| 使用API | `GET /admin/inventory-items`（在庫レベル取得） |

**表示内容:**
- 在庫残数が閾値以下（デフォルト: 5個）の商品を警告リスト表示
- 各商品名、SKU、現在の在庫数、閾値を表示
- 商品名クリックで商品編集ページへ遷移
- 在庫切れ（0個）の商品は赤色ハイライト

**実装方針:**
- 閾値はウィジェット内の設定で変更可能にする
- バッジ表示でアラート件数をトップページに表示

##### ③ 商品並び順・特集管理
| 項目 | 内容 |
|---|---|
| 配置場所 | 商品一覧ページ / コレクション詳細ページにウィジェットとして追加 |
| 優先度 | P2（あると良い） |
| 使用API | `PUT /admin/products`（メタデータ更新）, `PUT /admin/collections` |

**表示内容:**
- ドラッグ&ドロップで商品の表示順を変更
- 「トップページ特集に追加」トグルボタン
- コレクション内の商品並び替え

**実装方針:**
- 並び順はProductのメタデータ（`metadata.sort_order`）で管理する
- ストアフロント側では `sort_order` でソートして表示する
- ドラッグ&ドロップ: @dnd-kit/core を使用

##### ④ 注文ステータス一括変更
| 項目 | 内容 |
|---|---|
| 配置場所 | 注文一覧ページにウィジェットとして追加 |
| 優先度 | P2（あると良い） |
| 使用API | `POST /admin/orders/:id/fulfillments` 等（バッチ処理） |

**表示内容:**
- 注文一覧のチェックボックスで複数選択
- 一括ステータス変更ボタン（「選択した注文を出荷済みにする」等）
- 一括処理の確認ダイアログ
- 処理結果のサマリー表示（成功/失敗件数）

**実装方針:**
- バッチ処理はフロント側でPromise.allSettledを使用して並行実行
- 進捗バー表示（処理中の件数 / 全件数）
- エラーが発生した注文はリストで報告する

### 3.3 非機能要件

#### パフォーマンス
- Core Web Vitals目標: LCP < 2.5s, FID < 100ms, CLS < 0.1
- 商品一覧ページの初期表示: 1.5秒以内
- 商品画像: WebP/AVIF形式、next/imageで最適化
- Server Componentsによるサーバーサイドレンダリングの活用

#### アクセシビリティ
- WCAG 2.2 AA準拠
- キーボードナビゲーション対応
- スクリーンリーダー対応（適切なaria属性）
- カラーコントラスト比 4.5:1 以上

#### レスポンシブ対応
- モバイルファースト設計
- ブレイクポイント: sm(640px) / md(768px) / lg(1024px) / xl(1280px)

#### SEO
- 各ページのメタタグ動的生成（generateMetadata）
- 構造化データ（JSON-LD: Product, BreadcrumbList）
- サイトマップ自動生成
- OGP画像設定

#### 多言語対応（日本語 + 英語）
- Next.js App Routerのi18nルーティングを使用
- URLパス方式: `/ja/products/...`, `/en/products/...`
- デフォルト言語: 日本語（`/ja` は省略可）
- 翻訳管理: `next-intl` を使用
- 翻訳ファイル: `messages/ja.json`, `messages/en.json`
- 商品データの多言語化: Medusaのメタデータ（`metadata.title_en`, `metadata.description_en`）で管理
- UI文言（ボタン、ラベル、エラーメッセージ等）はすべて翻訳キーで管理
- 言語切替: ヘッダーに言語セレクター（🇯🇵 / 🇬🇧）を配置

| ファイルストレージ | S3互換 | - | 商品画像（Cloudflare R2 or AWS S3） |
| デプロイ（フロント） | Vercel | - | Edge Runtime対応 |
| デプロイ（バックエンド） | Railway | - | Medusa + PostgreSQL + Redis |
| テスト | Vitest + Playwright | - | ユニット + E2E |
| 型チェック | TypeScript | 5.x | strict mode |
| リンター | ESLint + Prettier | - | コード品質 |

### 4.3 Medusa SDK 使用方針

#### Store API（フロント向け）

```typescript
// lib/medusa.ts — SDK初期化（シングルトン）
import Medusa from "@medusajs/js-sdk"

export const sdk = new Medusa({
  baseUrl: process.env.NEXT_PUBLIC_MEDUSA_BACKEND_URL!,
  auth: { type: "session" },
})
```

**Server Components でのデータ取得（推奨パターン）:**
```typescript
// app/products/page.tsx
export default async function ProductsPage() {
  const { products, count } = await sdk.store.product.list({
    limit: 12,
    offset: 0,
    fields: "+variants.calculated_price",
  })
  return <ProductGrid products={products} />
}
```

**Client Components でのミューテーション:**
```typescript
// hooks/use-cart.ts
export function useAddToCart() {
  const queryClient = useQueryClient()
  return useMutation({
    mutationFn: ({ cartId, variantId, quantity }: AddToCartInput) =>
      sdk.store.cart.createLineItem(cartId, {
        variant_id: variantId,
        quantity,
      }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["cart"] })
    },
  })
}
```

#### Admin API（管理画面向け）

管理画面はMedusa Admin UIが標準提供する。カスタムウィジェットからAdmin APIを呼び出す場合は、Admin UI SDKのフックを使用する。

#### 認証フロー

```
[ログイン画面]
    ↓ POST /auth/customer/emailpass { email, password }
[Medusa Backend]
    ↓ セッションCookie発行
[ブラウザ]
    ↓ Cookie自動送信
[認証済みAPI呼び出し]
    → GET /store/customers/me
    → GET /store/customers/me/orders
```

cart_idはCookieに保存し、認証前のカートをログイン後に顧客に紐付ける。

#### Stripe テストモード方針

本プロジェクトではStripeを**テストモード専用**で使用し、実際の課金は発生しない。

**実装ルール:**
- 環境変数に設定するのはStripeの**テスト用APIキー**（`sk_test_xxx` / `pk_test_xxx`）のみ
- 本番用キー（`sk_live_xxx`）は絶対に設定しない
- チェックアウト完了画面に「これはデモ注文です。実際の課金は発生していません。」と明示する
- Stripe Elementsのカード入力フォームはテストモードで正常動作する

**テスト用カード番号:**
- 成功: `4242 4242 4242 4242`（有効期限: 任意の未来日、CVC: 任意の3桁）
- 失敗: `4000 0000 0000 0002`（カード拒否テスト用）

**本番移行時の対応:**
- 環境変数を本番用キーに差し替えるだけで実決済に切り替え可能
- チェックアウト完了画面のデモ表示を削除する
- Webhookエンドポイントを本番用に更新する

## 7. 開発フェーズ

### Phase 1: MVP（最小実用製品）

**期間目安:** 4〜6週間
**目標:** 商品を閲覧し、カートに入れて購入できる最小限のフロー

**含める機能（P0）:**
- Medusaバックエンドのセットアップ（商品・カート・注文・決済モジュール）
- シードスクリプト（ダミー商品20〜30点 + カテゴリ + プレースホルダー画像）
- PlaceholderImageコンポーネント（全画像をプレースホルダーで表示）
- トップページ（ヒーローバナー + 新着商品）
- 商品一覧（カテゴリフィルター付き）
- 商品詳細ページ
- カート機能（追加・数量変更・削除）
- チェックアウト（配送先入力 + Stripe決済）
- 注文完了画面
- レスポンシブ対応（モバイル + デスクトップ）
- 基本的なSEO設定（メタタグ + OGP）

**完了定義:**
- 商品の閲覧→カート追加→チェックアウト→注文完了のフローが動作する
- Stripeテストモードでの決済が成功する（テスト用カード番号使用）
- プレースホルダー画像・ダミーテキストで全画面が表示される
- モバイル・デスクトップ両方でレイアウトが崩れない
- Lighthouseパフォーマンススコア 80以上

### Phase 2: コア機能拡充

**期間目安:** 3〜4週間
**目標:** ユーザーアカウント機能とショップの完成度向上

**含める機能（P1）:**
- ユーザー認証（登録・ログイン・ログアウト）
- マイページ（プロフィール・住所管理・注文履歴）
- 多言語対応（next-intl による日本語 + 英語切替、翻訳ファイル整備）
- 検索機能
- カテゴリページ
- 商品詳細の充実（関連商品、画像ギャラリー改善）
- カートドロワー（ヘッダーからのクイックアクセス）
- 静的ページ（About、配送ポリシー、お問い合わせ）
- アクセシビリティ改善（WCAG 2.2 AA準拠）

**完了定義:**
- ユーザーが会員登録し、注文履歴を確認できる
- 日本語・英語の切り替えが全ページで正常動作する
- 検索とカテゴリフィルターが正常動作する
- WCAG 2.2 AA準拠の主要項目をクリア

### Phase 3: 改善・拡張

**期間目安:** 3〜4週間
**目標:** 運用に必要な管理機能とユーザー体験の向上

**含める機能（P2 + 改善）:**
- 管理画面カスタムウィジェット4機能
  - 売上・注文ダッシュボード（Rechartsでグラフ表示）
  - 在庫アラート（閾値設定付き）
  - 商品並び順・特集管理（ドラッグ&ドロップ）
  - 注文ステータス一括変更（バッチ処理）
- 商品レビュー機能
- ウィッシュリスト
- パフォーマンス最適化（Core Web Vitals全項目Green）
- アニメーション・マイクロインタラクション追加
- E2Eテスト整備（主要フロー）
- 本番環境へのデプロイ・ドメイン設定

**完了定義:**
- 管理画面で4つのカスタムウィジェットが表示・動作する
- Core Web Vitals全項目でGood判定
- 主要フローのE2Eテストがパスする
